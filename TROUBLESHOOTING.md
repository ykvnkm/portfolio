# Решение проблемы "Страница не найдена" на GitHub Pages

## Шаг 1: Проверьте настройки GitHub Pages

1. Откройте ваш репозиторий на GitHub: https://github.com/ykvnkm/portfolio
2. Перейдите в **Settings** (Настройки)
3. В левом меню найдите раздел **Pages**
4. Проверьте настройки:
   - **Source** (Источник): должно быть `Deploy from a branch`
   - **Branch** (Ветка): должно быть `gh-pages` и папка `/ (root)`
   - Если там написано "None" или другая ветка — выберите `gh-pages` и нажмите **Save**

## Шаг 2: Подождите несколько минут

После настройки GitHub Pages может потребоваться 1-5 минут для активации сайта.

## Шаг 3: Проверьте правильный URL

Ваш сайт должен быть доступен по адресу:
- **https://ykvnkm.github.io/portfolio**

**Важно:** 
- URL чувствителен к регистру
- Убедитесь, что используете правильный username (`ykvnkm`)
- Убедитесь, что используете правильное имя репозитория (`portfolio`)

## Шаг 4: Очистите кэш браузера

Иногда браузер кэширует старую версию. Попробуйте:
- Открыть сайт в режиме инкогнито
- Или очистить кэш браузера (Ctrl+Shift+Delete / Cmd+Shift+Delete)

## Шаг 5: Проверьте консоль браузера

1. Откройте сайт: https://ykvnkm.github.io/portfolio
2. Нажмите F12 (или Cmd+Option+I на Mac)
3. Перейдите на вкладку **Console**
4. Посмотрите, есть ли ошибки (красные сообщения)

## Шаг 6: Передеплойте проект (если нужно)

Если проблема не решена, попробуйте передеплоить:

```bash
npm run deploy
```

После этого подождите 2-3 минуты и проверьте сайт снова.

## Шаг 7: Проверьте содержимое ветки gh-pages

Убедитесь, что в ветке `gh-pages` есть файл `index.html` в корне:

1. На GitHub перейдите в ваш репозиторий
2. Нажмите на выпадающий список с ветками (обычно там написано "main")
3. Выберите ветку `gh-pages`
4. Убедитесь, что видите файл `index.html` в корне

## Частые проблемы:

### Проблема: "404 File not found"
**Решение:** Проверьте настройки GitHub Pages (Шаг 1) — скорее всего не выбрана ветка `gh-pages`

### Проблема: Страница загружается, но без стилей/изображений
**Решение:** Это проблема с путями. Убедитесь, что все пути относительные (без `/` в начале)

### Проблема: Сайт показывает README вместо вашего сайта
**Решение:** GitHub Pages не настроен или выбрана неправильная ветка

## Если ничего не помогло:

1. Проверьте, что репозиторий называется именно `portfolio` (не `Portfolio` или `Portfolio-`)
2. Проверьте, что ваш GitHub username — `ykvnkm`
3. Убедитесь, что репозиторий публичный (не приватный)
4. Попробуйте открыть прямой URL к файлу: https://ykvnkm.github.io/portfolio/index.html
